# Rust 区块链 - 简易加密货币

一个用 Rust 从零实现的简易加密货币系统，包含工作量证明（PoW）共识、Ed25519 数字签名钱包、交易签名与验证、动态难度调整等功能。

## 功能特性

- **工作量证明（PoW）** — 基于 SHA-256 的挖矿算法，通过调整难度维持约 10 秒的出块时间
- **Ed25519 数字签名钱包** — 密钥对生成、签名与验签，钱包可持久化到磁盘
- **交易系统** — 支持签名转账、余额校验、双花防护
- **挖矿奖励** — 矿工获得 Coinbase 奖励，每 100 个区块奖励减半
- **动态难度调整** — 每 10 个区块根据实际出块速度自动调整难度（范围 2~6）
- **链验证** — 完整校验所有区块哈希、PoW、交易签名，并独立重算账户余额
- **JSON 持久化** — 区块链状态保存到 `blockchain.json`，钱包保存到 `wallet.json`，重启后自动加载
- **中文交互式 CLI** — 菜单式操作界面，支持创建钱包、挖矿、转账、查询余额等功能

## 快速开始

### 环境要求

- Rust 工具链（[rustup](https://rustup.rs/)）1.85+

### 编译 & 运行

```bash
# 编译（release 模式性能更好）
cargo build --release

# 运行
cargo run --release
```

首次运行会自动创建创世区块。此时还没有钱包，选择菜单选项 `1` 创建或加载钱包。

### 交互式菜单

```

1. 创建/加载钱包
```

## 使用指南

### 1. 创建钱包

选择菜单 `1`，如果本地已有 `wallet.json` 则加载，否则自动生成新的 Ed25519 密钥对。钱包地址即为公钥的十六进制字符串。

### 2. 挖矿

选择菜单 `2`，将待确认交易打包成区块并执行 PoW 挖矿。Coinbase 奖励（初始 50 聪，每 100 块减半）归当前钱包所有。挖矿完成后区块链自动保存。

### 3. 转账

选择菜单 `3`，输入接收方地址和转账金额（聪，1 币 = 1 亿聪）。交易将用你的钱包签名后广播到待确认池中，需等待矿工打包确认。

### 4. 查询余额

选择菜单 `4`，输入地址（支持前缀匹配）查询任意账户余额，或直接回车查询当前钱包余额。

### 5. 验证整条链

选择菜单 `7`，系统会遍历整条链验证：
- 每个区块的哈希是否满足 PoW 要求
- 前后区块链接是否正确
- 所有交易的签名是否有效
- 独立重算的余额是否与存储一致

## 项目结构

```
src/
├── main.rs          # 交互式 CLI 入口
├── blockchain.rs    # 核心：区块、链、挖矿、验证、持久化
├── wallet.rs        # Ed25519 钱包：密钥生成、签名、持久化
└── transaction.rs   # 交易结构：创建、签名、验证
```

### 核心设计

- **账户模型** — 采用类账户模型，转账时立即从发送方余额扣除（防双花），挖矿后记入接收方余额
- **地址** — 地址即 Ed25519 公钥的十六进制字符串（64 字符）
- **Coinbase 交易** — 发送方为 `COINBASE`，无需签名，挖矿时自动生成
- **持久化** — 每次挖矿后保存链状态；待确认交易不持久化（重启丢失，与真实交易池行为一致）
- **难度调整** — 每 10 个区块计算平均出块时间，偏离 10 秒目标则 ±1 调整难度
